Build steps

Follow these steps in order. Do not skip ahead ‚Äî each step depends on the last.

Step 1 ‚Äî Set up your project

  1. Create a new Scratch project and rename it: Y8 Breakout ‚Äì Your Name.
  2. Choose a simple background.
  3. Create sprites: Paddle, Ball, Brick (or make a brick costume).

Tip

Keep costumes simple at first. Fancy graphics can come later.

Step 2 ‚Äî Paddle movement

  • Use forever to keep checking input.
  • If left key pressed ‚Üí change x by negative amount.
  • If right key pressed ‚Üí change x by positive amount.
  • Add boundaries so it can‚Äôt leave the screen.

Step 3 ‚Äî Ball movement + bouncing

  • Start ball on the paddle (optional) then launch with space.
  • Use a direction variable or set direction.
  • Move a small amount each tick (e.g. move 6 steps).
  • Use if on edge, bounce.
  • When touching paddle ‚Üí bounce (and optionally change angle).

Quick check

Q: Why is moving 4‚Äì8 steps smoother than moving 20 steps?

Step 4 ‚Äî Bricks

  • Create a brick sprite and make it look like a rectangle.
  • Use clones to generate a grid of bricks.
  • Make rows/columns using nested loops (repeat inside repeat).

Step 5 ‚Äî Brick collisions

  • If ball touches a brick clone ‚Üí delete that clone.
  • Make the ball bounce when it hits a brick (reverse direction).
  • Add score +1 per brick.

Step 6 ‚Äî Lives, reset, win/lose

  • Create variables: score, lives.
  • If ball touches bottom edge ‚Üí lose a life and reset ball.
  • If lives reaches 0 ‚Üí Game Over.
  • If all bricks are gone ‚Üí You Win.

Minimum viable game (must have)

  • Paddle controlled by keys or mouse
  • Ball bounces off walls + paddle
  • Bricks disappear when hit
  • Score + win/lose condition

Feature ideas

Once your minimum viable game works, add features to improve it. Pick from the tiers below.

Core (do these first)

  • Start screen with instructions (press space to start).
  • Smoother bounce: change angle depending on where it hits the paddle.
  • Sound effects: wall bounce, brick break, lose life.
  • Brick varieties: different colours = different points.

Secure (good challenge)

  • Levels: new brick layout each level.
  • Increasing difficulty: ball speed increases each level.
  • Lives UI: show hearts or icons rather than a number.
  • Pause: press P to pause/unpause.

Stretch (for top scores)

  • Power-ups: wider paddle, multi-ball, slow motion.
  • Brick durability: bricks take 2‚Äì3 hits (use a variable per clone).
  • Combo scoring: bonus points for fast brick streaks.
  • Boss level: moving bricks or a moving ‚Äútarget‚Äù to hit.
  • High score: store best score (Scratch cloud var optional).

How to add a feature without breaking everything

  1. Duplicate your project first (File ‚Üí Save as a copy).
  2. Add one small change.
  3. Test it for 60 seconds.
  4. If it breaks, undo and try a simpler version.

Common problems (quick fixes)

  • Ball goes through bricks: reduce speed (steps) or check collisions more often.
  • Paddle leaves screen: clamp x position (min/max).
  • Bricks won‚Äôt delete: make sure you‚Äôre deleting the brick clone, not the ball.
